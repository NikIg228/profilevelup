✅ CURSOR PROMPT — Mobile-first Intro Overlay (≤ 5s) + Skip → Hero
0) Контекст

Проект: лендинг/сайт.
Требование: при первом открытии или повторном входе пользователь сначала видит полноэкранную интро-анимацию (pain phrases).
Пока интро активно — на странице НЕ должно быть видно НИ ОДНОГО компонента/контента (header/hero/секции).
После интро (или по тапу skip) — показывается обычная главная страница.

Ограничение по времени: интро максимум 5 секунд (включая переход к светлому фону и показ контента).
Приоритет: mobile-first (iOS/Android). Десктоп повторяет логику, но вторичен.

1) Что нужно реализовать (сущности)

Компонент/слой IntroOverlay (full-screen overlay поверх всего).

Таймлайн анимации 3 фраз (по очереди).

Мгновенный Skip по тапу/клику в любой момент.

Контроль рендера: пока интро активно — основной контент вообще не рендерить (или рендерить, но полностью скрывать так, чтобы 100% ничего не видно и не кликается; предпочтительно — не рендерить для чистоты).

Сохранение флага показа (localStorage) — чтобы интро показывался по заданной логике.

2) Жёсткая логика показа (localStorage)
Вариант по умолчанию (рекомендованный):

Показ интро:

если нет localStorage.intro_seen = "1"

ИЛИ пользователь заходит повторно (но тогда показываем не всегда, иначе раздражает)

Нужная логика (выбери одну и реализуй):

A) Только 1 раз на устройстве (самый безопасный UX):

Если intro_seen отсутствует → показать интро, затем записать intro_seen="1"

Если есть → интро не показывать

B) 1 раз в сутки (если хочешь иногда возвращать “крючок”):

intro_last_shown = timestamp

Если прошло > 24 часа → показать интро и обновить timestamp

✅ Реализуй A по умолчанию, но кодом предусмотрим легко переключить на B.

3) Таймлайн анимаций (≤ 5.0s TOTAL)

3 фразы показываются по очереди, строго по центру.

Фразы (точно такие):

«Выбери нормальную профессию!»

«На этом денег не заработаешь!»

«Какие таланты? Главное — диплом!»

Тайминг (очень конкретно):

Общая длительность ≤ 5.0s включая fade-to-light и появление контента.

Для каждой фразы — 1.35s:

pop-in: 0.18s

opacity: 0 → 1

scale: 0.92 → 1.00

easing: easeOut

shake-light: 0.80s

лёгкая вибрация по X (амплитуда 4–6px максимум)

без rotation

fade-out: 0.25s

opacity: 1 → 0

scale: 1.00 → 0.98 (едва заметно)

gap: 0.12s пауза (чтобы не “слипалось”)

Итого на 3 фразы:
1.35s × 3 = 4.05s

Переход fade-to-light:

0.55s

Фон: #050B1C → #F6F8FF (или близкий светлый)

Одновременно overlay растворяется (opacity overlay: 1 → 0)

TOTAL: 4.05 + 0.55 = 4.6s ✅
(Запас до 5s есть)

4) Skip UX (обязательно)

Пользователь может тапнуть по экрану в любой момент, чтобы пропустить интро.

Правила Skip:

Tap/click по любому месту overlay → мгновенно:

остановить все таймеры/анимации

мгновенно выполнить финальный переход:

overlay → opacity 0 за 150–200ms

показать основной контент (hero) сразу

записать флаг intro_seen

Доп. доступность:

Также пропуск по Escape (для десктопа)

aria-label="Skip intro" на overlay, role="button"

5) Центровка и мобильная типографика (строго)

Overlay:

position: fixed; inset: 0;

display: grid; place-items: center;

padding: 24px;

text-align: center;

touch-action: manipulation;

Текст:

Mobile font-size: clamp(24px, 6vw, 28px)

font-weight: 800 (или 700 минимум)

line-height: 1.35–1.5

max-width: 20–24ch (чтобы фразы не растягивались на огромную ширину)

color: #F2F4FF

Фон:

initial: #050B1C

fade-to-light: #F6F8FF

6) Производительность (60 FPS)

Запрещено:

тяжёлые blur/backdrop-filter

box-shadow на большом тексте

сложные фильтры

огромные SVG/видео внутри интро

Разрешено (и нужно):

анимации только transform и opacity

will-change: transform, opacity

не более 1 элемента текста на экране одновременно

7) Архитектура (чтобы Cursor не накосячил)

Реализуй так:

Стейт на уровне страницы/лейаута:

const [showIntro, setShowIntro] = useState<boolean>(...)

Пока showIntro === true:

рендерим ТОЛЬКО <IntroOverlay onFinish={...} />

основной контент не рендерим

Когда showIntro === false:

рендерим обычный лейаут/главную страницу

Это гарантирует “НЕ показывается ни один компонент/контент”.

8) Реализация анимации (выбери один подход и сделай идеально)
Предпочтительный вариант: Framer Motion

Если проект React и уже есть Framer Motion:

Используй motion.div для текста

Таймлайн управляй состоянием index (0..2) + phase

Не делай сложные nested animation chains без контроля cleanup

Альтернатива: чистый CSS + JS таймеры

CSS keyframes: popIn, shakeLight, fadeOut

JS меняет текст + классы

Обязательно cleanup таймеров при unmount и при skip

✅ Важно: таймеры должны очищаться (иначе баги при быстрых переходах).

9) Псевдокод поведения (как должно работать)

На mount страницы:

если intro_seen нет → showIntro=true

иначе showIntro=false

Если showIntro=true:

показываем overlay

запускаем таймлайн фраз

По окончанию фразы №3:

запускаем fade-to-light 0.55s

затем setShowIntro(false)

По тапу:

stop timeline

быстро скрываем overlay (0.2s)

setShowIntro(false)

10) Definition of Done (что считать “готово”)

 На мобильном при первом заходе показывается интро

 Во время интро НЕ видно header/hero/секций

 3 фразы появляются по очереди с pop-in + shake + fade-out

 Всё укладывается в ≤ 5 сек

 Тап в любой момент пропускает интро мгновенно

 После интро появляется обычная главная страница с hero

 Никаких лагов: transform/opacity, 60 FPS

 Нет утечек таймеров (cleanup)

 Повторный вход не показывает интро (если логика A)

11) Важные мелочи (чтобы не было фейлов)

Overlay должен быть выше всего: z-index: 9999

Пока overlay активен:

document.body.style.overflow = "hidden"

затем вернуть как было

На iOS:

использовать height: 100dvh (или fallback)

Не использовать position: absolute внутри — только фиксированный контейнер

12) Что НЕ делать (частые ошибки Cursor)

Не рендерить сайт под overlay “чуть-чуть видно” (нельзя)

Не растягивать анимацию >5 секунд

Не запускать три фразы одновременно

Не забывать cleanup таймеров

Не добавлять лишние эффекты
