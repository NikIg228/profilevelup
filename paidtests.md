Ты работаешь с frontend-системой тестирования личности.

ВАЖНО:
В системе есть ДВЕ РАЗНЫЕ логики тестов:
1) FREE — уже реализована, её трогать НЕЛЬЗЯ
2) EXTENDED и PREMIUM — новая логика, одинаковая для обоих тарифов

⚠️ ВАЖНО ПО НАЗВАНИЯМ В UI:
- EXTENDED в интерфейсе называется: «Личный разбор»
- PREMIUM в интерфейсе называется: «Подросток и родитель»

Это РАЗНЫЕ тарифы, но с ОДИНАКОВОЙ логикой теста.

========================
1️⃣ FREE (НЕ ТРОГАТЬ)
========================
FREE-тесты:
- состоят из 5 вопросов
- используют positional resultMapping
- возвращают resultIndex вида:
  [E/I][N/S][T/F][J/P/W]
- логика формирования результата уже существует
- код FREE тестов и engine менять запрещено

========================
2️⃣ EXTENDED / PREMIUM (НОВАЯ ЛОГИКА)
========================

Для тарифов EXTENDED («Личный разбор») и PREMIUM («Подросток и родитель»)
необходимо реализовать ОТДЕЛЬНУЮ логику подсчёта результата.

📌 Структура теста:
- всего 28 вопросов
- 4 блока по 7 вопросов
- каждый блок отвечает за одну дихотомию:

Блок 1 → E / I
Блок 2 → S / N
Блок 3 → T / F
Блок 4 → J / P

Каждый вопрос:
- имеет 2 варианта ответа
- вариант A → первая буква дихотомии
- вариант B → вторая буква дихотомии

========================
3️⃣ Подсчёт результата
========================

Для каждого блока:
- посчитать количество ответов A и B
- формат: A_count / B_count

Правило определения итоговой буквы:

Если |A_count - B_count| === 1
→ использовать СПЕЦИАЛЬНУЮ (срединную) букву

Иначе
→ использовать букву стороны большинства

Специальные буквы:
- E / I → Z
- S / N → X
- T / F → Q
- J / P → W

========================
4️⃣ Формирование итогового типа
========================

Итоговый resultIndex формируется строго в порядке MBTI:

[E/I/Z] + [S/N/X] + [T/F/Q] + [J/P/W]

Пример:
E + X + T + P = EXTP

========================
5️⃣ Требования к реализации
========================

- логика EXTENDED / PREMIUM должна быть изолирована от FREE
- UI не знает формул подсчёта
- UI получает только resultIndex
- EXTENDED («Личный разбор») и PREMIUM («Подросток и родитель»)
  используют ОДНУ И ТУ ЖЕ логику подсчёта
- backend НЕ реализуется здесь
- backend получит только resultIndex и пользовательские данные
- система должна поддерживать все комбинации,
  включая срединные (Z/X/Q/W)

========================
6️⃣ Псевдокод (обязателен к использованию)
========================

function getLetter(A_count, B_count, primary, secondary, middle) {
  if (Math.abs(A_count - B_count) === 1) {
    return middle;
  }
  return (A_count > B_count) ? primary : secondary;
}

type =
  getLetter(E_A, E_B, 'E', 'I', 'Z') +
  getLetter(S_A, S_B, 'S', 'N', 'X') +
  getLetter(T_A, T_B, 'T', 'F', 'Q') +
  getLetter(J_A, J_B, 'J', 'P', 'W');
