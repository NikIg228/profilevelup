# Инструкция по настройке Supabase для авторизации

Это пошаговая инструкция по настройке Supabase для работы авторизации и регистрации через email + пароль.

## Шаг 1: Создание проекта в Supabase

1. Перейдите на https://supabase.com
2. Войдите в свой аккаунт или зарегистрируйтесь
3. Нажмите "New Project"
4. Заполните форму:
   - **Name**: Название проекта (например, "ProfiLevelUp")
   - **Database Password**: Придумайте надежный пароль (сохраните его!)
   - **Region**: Выберите ближайший регион
5. Нажмите "Create new project"
6. Дождитесь завершения создания проекта (обычно 1-2 минуты)

## Шаг 2: Получение API ключей

1. В панели проекта перейдите в **Settings** → **API**
2. Найдите раздел **Project API keys**
3. Скопируйте следующие значения:
   - **Project URL** (например: `https://xxxxx.supabase.co`)
   - **anon public** ключ (начинается с `eyJ...`)

⚠️ **ВАЖНО**: Используйте именно **anon public** ключ, а не **service_role** ключ!

## Шаг 3: Настройка переменных окружения

1. В корне проекта создайте файл `.env` (если его нет)
2. Добавьте следующие строки:

```env
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

3. Замените значения на ваши из шага 2
4. Сохраните файл
5. Перезапустите dev-сервер (`npm run dev`)

## Шаг 4: Настройка базы данных

1. В панели Supabase перейдите в **SQL Editor**
2. Нажмите **New query**
3. Откройте файл `supabase_setup.sql` из проекта
4. Скопируйте весь содержимое файла
5. Вставьте в SQL Editor
6. Нажмите **Run** (или `Ctrl+Enter`)
7. Дождитесь завершения выполнения
8. Проверьте, что в результатах нет ошибок

## Шаг 5: Настройка авторизации

### 5.1. Включение Email провайдера

1. Перейдите в **Authentication** → **Providers**
2. Найдите **Email** в списке провайдеров
3. Убедитесь, что Email включен (переключатель должен быть активен)

### 5.2. Настройка URL для редиректов

1. Перейдите в **Authentication** → **URL Configuration**
2. В поле **Site URL** укажите:
   - Для разработки: `http://localhost:5173`
   - Для продакшена: ваш домен (например, `https://yourdomain.com`)
3. В разделе **Redirect URLs** добавьте:
   - `http://localhost:5173/**` (для разработки)
   - `https://yourdomain.com/**` (для продакшена)
4. Нажмите **Save**

### 5.3. Настройка подтверждения email (опционально)

1. Перейдите в **Authentication** → **Email Templates**
2. Вы можете настроить шаблоны писем:
   - **Confirm signup** - письмо для подтверждения регистрации
   - **Reset password** - письмо для сброса пароля
   - **Change email address** - письмо для изменения email

**Рекомендация**: Для начала можно отключить подтверждение email:
1. Перейдите в **Authentication** → **Providers** → **Email**
2. Отключите **Confirm email** (если хотите, чтобы пользователи могли сразу входить после регистрации)

## Шаг 6: Проверка настройки

### 6.1. Проверка таблицы profiles

1. Перейдите в **Table Editor**
2. Должна быть видна таблица **profiles**
3. Таблица должна содержать колонки:
   - `id` (UUID)
   - `full_name` (text)
   - `email` (text)
   - `created_at` (timestamp)
   - `updated_at` (timestamp)

### 6.2. Проверка RLS политик

1. Перейдите в **Table Editor** → **profiles**
2. Нажмите на иконку щита (RLS)
3. Должны быть видны политики:
   - Users can view own profile
   - Users can update own profile
   - Users can insert own profile
   - Users can delete own profile

### 6.3. Проверка триггеров

1. Перейдите в **Database** → **Functions**
2. Должны быть видны функции:
   - `handle_new_user`
   - `handle_updated_at`
   - `handle_email_change`

## Шаг 7: Тестирование

1. Запустите проект: `npm run dev`
2. Откройте http://localhost:5173/account
3. Попробуйте зарегистрироваться:
   - Введите email и пароль
   - Нажмите "Зарегистрироваться"
   - Если включено подтверждение email - проверьте почту
4. Попробуйте войти:
   - Введите email и пароль
   - Нажмите "Войти"
5. Проверьте личный кабинет:
   - Должна отображаться информация о пользователе
   - Кнопка "Выйти" должна работать

## Возможные проблемы

### Проблема: "Missing Supabase environment variables"

**Решение:**
- Убедитесь, что файл `.env` создан в корне проекта
- Проверьте, что переменные названы правильно: `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`
- Перезапустите dev-сервер после создания/изменения `.env`

### Проблема: "Invalid API key"

**Решение:**
- Убедитесь, что используете **anon public** ключ, а не **service_role**
- Anon ключ начинается с `eyJ...`
- Service role ключ начинается с `sb_secret_` (его нельзя использовать в браузере!)

### Проблема: "User already registered"

**Решение:**
- Пользователь с таким email уже существует
- Попробуйте войти или использовать другой email

### Проблема: "Email not confirmed"

**Решение:**
- Проверьте почту и перейдите по ссылке подтверждения
- Или отключите подтверждение email в настройках Supabase

### Проблема: CORS ошибки

**Решение:**
- Убедитесь, что в **Authentication** → **URL Configuration** добавлены правильные URL
- Для разработки: `http://localhost:5173`
- Проверьте, что Site URL указан правильно

## Дополнительные настройки

### Настройка SMTP (для отправки писем)

По умолчанию Supabase использует встроенный SMTP. Для продакшена рекомендуется настроить свой SMTP:

1. Перейдите в **Settings** → **Auth** → **SMTP Settings**
2. Настройте SMTP провайдера (Gmail, SendGrid, Mailgun и т.д.)
3. Введите данные для подключения
4. Сохраните настройки

### Настройка кастомных шаблонов писем

1. Перейдите в **Authentication** → **Email Templates**
2. Выберите шаблон (Confirm signup, Reset password и т.д.)
3. Настройте HTML и текст письма
4. Используйте переменные:
   - `{{ .ConfirmationURL }}` - ссылка для подтверждения
   - `{{ .Email }}` - email пользователя
   - `{{ .SiteURL }}` - URL сайта
