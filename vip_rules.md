# Техническое задание: Логика формирования типа личности (VIP/EXTENDED/PREMIUM тест)

## 1️⃣ Структура теста

### Общая информация
- **Всего вопросов:** 28
- **Формат ответов:** A или B
- **Количество блоков:** 4
- **Вопросов в блоке:** 7

### Распределение вопросов по блокам

| Блок | Дихотомия | Обозначения | Номера вопросов | Маппинг ответов |
|------|-----------|-------------|-----------------|-----------------|
| 1 | E / I | Экстраверсия / Интроверсия | 1, 5, 9, 13, 17, 21, 25 | A = E, B = I |
| 2 | S / N | Сенсорика / Интуиция | 2, 6, 10, 14, 18, 22, 26 | A = S, B = N |
| 3 | T / F | Логика / Эмоции | 3, 7, 11, 15, 19, 23, 27 | A = T, B = F |
| 4 | J / P | Организованность / Гибкость | 4, 8, 12, 16, 20, 24, 28 | A = J, B = P |

### Правило маппинга ответов
- **A** → первая буква дихотомии (E, S, T, J)
- **B** → вторая буква дихотомии (I, N, F, P)

## 2️⃣ Подсчёт результата по каждому блоку

### Алгоритм подсчёта
Для каждого блока (E/I, S/N, T/F, J/P):
1. Подсчитывается количество ответов **A** и **B**
2. Формат результата: `A_count.B_count` (например: `5.2`, `4.3`, `7.0`)

### Примеры подсчёта
- Блок E/I: ответы [A, A, B, A, A, B, A] → результат `5.2` (5 ответов A, 2 ответа B)
- Блок S/N: ответы [A, A, A, B, A, B, B] → результат `4.3` (4 ответа A, 3 ответа B)

## 3️⃣ Правило определения итоговой буквы для каждой дихотомии

### Таблица результатов

| A_count | B_count | Разница | Итоговая буква | Интерпретация |
|---------|---------|---------|----------------|---------------|
| 7 | 0 | 7 | Первая буква | Выраженный тип |
| 6 | 1 | 5 | Первая буква | Выраженный тип |
| 5 | 2 | 3 | Первая буква | Слабовыраженный тип |
| 4 | 3 | 1 | **Срединная буква** | Тип не выражен |
| 3 | 4 | 1 | **Срединная буква** | Тип не выражен |
| 2 | 5 | 3 | Вторая буква | Слабовыраженный тип |
| 1 | 6 | 5 | Вторая буква | Выраженный тип |
| 0 | 7 | 7 | Вторая буква | Выраженный тип |

### Специальные буквы для срединных значений

| Дихотомия | Первая буква | Вторая буква | Срединная буква |
|-----------|--------------|--------------|-----------------|
| E / I | E | I | **Z** |
| S / N | S | N | **X** |
| T / F | T | F | **Q** |
| J / P | J | P | **W** |

### Формула преобразования

```typescript
function getLetter(
  aCount: number, 
  bCount: number, 
  primary: string,    // Первая буква (E, S, T, J)
  secondary: string,  // Вторая буква (I, N, F, P)
  middle: string     // Срединная буква (Z, X, Q, W)
): string {
  if (Math.abs(aCount - bCount) === 1) {
    return middle; // Срединный тип
  }
  return (aCount > bCount) ? primary : secondary;
}
```

### Пример расчёта

**Входные данные:**
- Блок E/I: `5.2` → E (5 > 2, разница = 3)
- Блок S/N: `4.3` → X (разница = 1 → срединная буква)
- Блок T/F: `7.0` → T (7 > 0, разница = 7)
- Блок J/P: `3.4` → P (3 < 4, разница = 1 → срединная буква, но 3 < 4, поэтому P)

**Итоговый тип:** `EXTP`

## 5️⃣ Подсчёт 7 осей (все шкалы 0–100)

### Ось 1: Социальный режим (0 Автономно ↔ 100 Через людей)

**Источник:** Блок E/I (вопросы 1, 5, 9, 13, 17, 21, 25)

```typescript
energy_A = count(answer[q] === "A" for q in [1,5,9,13,17,21,25])
AXIS_1_ENERGY = round(energy_A / 7 * 100)
```

- A = E = "через людей" → увеличивает значение
- B = I = "автономно" → уменьшает значение

### Ось 2: Фокус мышления (0 Конкретика и факты ↔ 100 Идеи и сценарии)

**Источник:** Блок S/N (вопросы 2, 6, 10, 14, 18, 22, 26)

```typescript
focus_B = count(answer[q] === "B" for q in [2,6,10,14,18,22,26])
AXIS_2_FOCUS = round(focus_B / 7 * 100)
```

- A = S = "конкретика" → уменьшает значение
- B = N = "идеи/сценарии" → увеличивает значение

### Ось 3: Основание решений (0 Логика/справедливость ↔ 100 Люди/ценности)

**Источник:** Блок T/F (вопросы 3, 7, 11, 15, 19, 23, 27)

```typescript
decisions_B = count(answer[q] === "B" for q in [3,7,11,15,19,23,27])
AXIS_3_DECISIONS = round(decisions_B / 7 * 100)
```

- A = T = "логика/справедливость" → уменьшает значение
- B = F = "люди/ценности" → увеличивает значение

### Ось 4: Стиль организации (0 Адаптация ↔ 100 План/структура)

**Источник:** Блок J/P (вопросы 4, 8, 12, 16, 20, 24, 28)

```typescript
structure_A = count(answer[q] === "A" for q in [4,8,12,16,20,24,28])
AXIS_4_STRUCTURE = round(structure_A / 7 * 100)
```

- A = J = "план/структура" → увеличивает значение
- B = P = "адаптация" → уменьшает значение

### Ось 5: Драйвер мотивации (0 Результат ↔ 100 Смысл)

**Вычисляется из осей 1–4** (без новых вопросов)

```typescript
AXIS_5_MOTIVATION = round(
  0.45 * AXIS_2_FOCUS + 
  0.35 * AXIS_3_DECISIONS + 
  0.20 * (100 - AXIS_4_STRUCTURE)

```

**Интерпретация:** "Смысл" растёт с идеями/сценариями, людьми/ценностями и гибкостью.

### Ось 6: Старт действий (0 Через план ↔ 100 Через пробу/эксперимент)

**Вычисляется из осей 1–4** (без новых вопросов)

```typescript
AXIS_6_START = round(
  0.80 * (100 - AXIS_4_STRUCTURE) + 
  0.20 * AXIS_2_FOCUS
)
```

**Интерпретация:** "Через пробу" растёт при низкой структурности и высоком фокусе на идеях.

### Ось 7: Стиль диалога в напряжении (0 Прямо/жёстко ↔ 100 Мягко/согласуя)

**Вычисляется из осей 1–4** (без новых вопросов)

```typescript
AXIS_7_CONFLICT = round(
  0.60 * AXIS_3_DECISIONS + 
  0.20 * (100 - AXIS_1_ENERGY) + 
  0.20 * (100 - AXIS_4_STRUCTURE)
)
```

**Интерпретация:** "Мягкость/согласование" растёт с ориентацией на людей/ценности, интроверсией и гибкостью.

## 6️⃣ Метрики "Выраженность" и "Уверенность" (0–100)

### 6.1. Выраженность (EXPRESSION)

Показывает "насколько стиль заметный" (насколько оси 1–4 далеко от середины 50).

```typescript
EXPRESSION = round(
2 * average(
    abs(AXIS_1_ENERGY - 50),
    abs(AXIS_2_FOCUS - 50),
    abs(AXIS_3_DECISIONS - 50),
    abs(AXIS_4_STRUCTURE - 50)
  )
)
```

**Диапазон:** 0–100
- **0** = все оси в середине (50)
- **100** = все оси на краях (0 или 100)

### 6.2. Уверенность (CONFIDENCE)

Показывает согласованность ответов внутри каждой группы из 7 вопросов.

#### A) Сила перевеса (base_conf)

Для каждой группы (E/I, S/N, T/F, J/P):
1. Считаем `margin = abs(countA - countB)` (на 7 вопросах margin ∈ {1, 3, 5, 7})
2. Переводим в балл: `conf_margin = 30 + 10 * margin`
3. Усредняем по 4 группам

```typescript
conf_margin(margin) = 30 + 10 * margin

base_conf = average(
conf_margin(abs(E_countA - E_countB)),
conf_margin(abs(S_countA - S_countB)),
conf_margin(abs(T_countA - T_countB)),
conf_margin(abs(J_countA - J_countB))
)
```

#### B) Стабильность (stability)

Для каждой группы из 7 вопросов:
1. Берём первые 3 вопроса и последние 4 вопроса
2. Считаем долю "правого поля" в каждой части
3. Вычисляем разницу: `stability_group = 100 - abs(p1 - p2)`

**Определение "правого поля":**
- E/I: right = A (E)
- S/N: right = B (N)
- T/F: right = B (F)
- J/P: right = A (J)

```typescript
stability = average(
  stability_EI,
  stability_SN,
  stability_TF,
  stability_JP
)
```

#### Итоговая уверенность

```typescript
CONFIDENCE = round(0.70 * base_conf + 0.30 * stability)
```

**Диапазон:** 0–100
- **Высокая уверенность** = согласованные ответы внутри групп
- **Низкая уверенность** = противоречивые ответы

## 7️⃣ Выбор текстовых модулей (3 модуля, L/M/R)

### Правило диапазонов

В отчёт вставляются 3 динамических текста по осям 5–7:

| Ось | Название модуля | Зависит от |
|-----|-----------------|------------|
| 5 | MODULE_MOTIVATION | AXIS_5_MOTIVATION |
| 6 | MODULE_START | AXIS_6_START |
| 7 | MODULE_CONFLICT | AXIS_7_CONFLICT |

### Функция определения диапазона

```typescript
function getBand(score: number): 'L' | 'M' | 'R' {
  if (score <= 35) return 'L';
  if (score <= 64) return 'M';
  return 'R';
}
```

### Диапазоны значений

| Диапазон | Буква | Описание |
|----------|-------|----------|
| 0–35 | **L** | Левая сторона шкалы |
| 36–64 | **M** | Середина шкалы |
| 65–100 | **R** | Правая сторона шкалы |

### Хранение текстов модулей

Тексты модулей хранятся отдельно (JSON) по возрастным группам:

```json
{
  "ageGroups": {
    "12-17": {
      "motivation": {
        "L": "Текст для L...",
        "M": "Текст для M...",
        "R": "Текст для R..."
      },
      "start": { ... },
      "conflict": { ... }
    },
    "18-20": { ... },
    "21+": { ... }
  }
}
```

**Важно:** Мы храним всего 9 текстов (по 3 варианта на каждую ось) для каждой возрастной группы. Всего 3 возрастные группы × 9 текстов = 27 текстовых модулей, которые комбинируются автоматически.

## 🔟 Алгоритм генерации отчёта (псевдокод)

```typescript
function generateReport(
  answers: ExtendedAnswers,  // 28 ответов A/B
  userName: string,
  userGender: 'male' | 'female',
  userAge: number
): Report {
  // 1. Подсчёт результата по блокам
  const eiResult = countBlock(answers, [1,5,9,13,17,21,25]);
  const snResult = countBlock(answers, [2,6,10,14,18,22,26]);
  const tfResult = countBlock(answers, [3,7,11,15,19,23,27]);
  const jpResult = countBlock(answers, [4,8,12,16,20,24,28]);
  
  // 2. Определение итоговых букв
  const letter1 = getLetter(eiResult.aCount, eiResult.bCount, 'E', 'I', 'Z');
  const letter2 = getLetter(snResult.aCount, snResult.bCount, 'S', 'N', 'X');
  const letter3 = getLetter(tfResult.aCount, tfResult.bCount, 'T', 'F', 'Q');
  const letter4 = getLetter(jpResult.aCount, jpResult.bCount, 'J', 'P', 'W');
  
  const resultType = letter1 + letter2 + letter3 + letter4;
  
  // 3. Подсчёт 7 осей
  const axis1 = calculateAxis1(answers);
  const axis2 = calculateAxis2(answers);
  const axis3 = calculateAxis3(answers);
  const axis4 = calculateAxis4(answers);
  const axis5 = calculateAxis5(axis2, axis3, axis4);
  const axis6 = calculateAxis6(axis2, axis4);
  const axis7 = calculateAxis7(axis1, axis3, axis4);
  
  // 4. Подсчёт метрик
  const expression = calculateExpression(axis1, axis2, axis3, axis4);
  const confidence = calculateConfidence(eiResult, snResult, tfResult, jpResult);
  
  // 5. Определение возрастной группы
  // Функция getAgeGroup возвращает: '12-17' | '18-20' | '21+'
  function getAgeGroup(age: number): '12-17' | '18-20' | '21+' {
    if (age >= 12 && age <= 17) return '12-17';
    if (age >= 18 && age <= 20) return '18-20';
    return '21+';
  }
  const ageGroup = getAgeGroup(userAge);
  
  // 6. Выбор текстовых модулей
  const moduleMotivation = getModule('motivation', ageGroup, getBand(axis5));
  const moduleStart = getModule('start', ageGroup, getBand(axis6));
  const moduleConflict = getModule('conflict', ageGroup, getBand(axis7));
  
  // 7. Выбор базового шаблона
  const templateKey = `${resultType}_${userGender}_${ageGroup}`;
  const template = getTemplate(templateKey);
  
  // 8. Генерация отчёта
  const report = template.replace({
    USER_NAME: userName,
    USER_GENDER: userGender,
    USER_AGE: userAge,
    RESULT_TYPE: resultType,
    EXPRESSION: expression,
    CONFIDENCE: confidence,
    AXIS_1_ENERGY: axis1,
    AXIS_2_FOCUS: axis2,
    AXIS_3_DECISIONS: axis3,
    AXIS_4_STRUCTURE: axis4,
    AXIS_5_MOTIVATION: axis5,
    AXIS_6_START: axis6,
    AXIS_7_CONFLICT: axis7,
    MODULE_MOTIVATION: moduleMotivation,
    MODULE_START: moduleStart,
    MODULE_CONFLICT: moduleConflict
  });
  
  return report;
}
```

## 📝 Примечания

1. **Все вычисления должны быть точными** - используйте округление только в финальных значениях осей и метрик.

2. **Срединные значения** (Z/X/Q/W) используются только когда `|A_count - B_count| === 1`.

3. **Текстовые модули** выбираются автоматически на основе диапазонов осей 5-7, что позволяет создавать персонализированные отчёты без умножения количества шаблонов.

4. **База отчётов** должна поддерживать все 81 возможную комбинацию типов (включая срединные значения) для каждой комбинации пол+возрастная группа.

5. **Универсальный раздел** "Как читать цифры глубже / пояснения" добавляется в конец каждого отчёта и не зависит от типа личности.
