# 📋 Эталонная логика генерации результатов для FREE тестов

## 1️⃣ Структура бесплатного теста

**Всего вопросов:** 5

Каждый вопрос отвечает за один из параметров типа личности (аналог MBTI).

| № вопроса | Пара параметров | Возможные ответы | Примечание |
|-----------|----------------|------------------|------------|
| 1 | P / J | P или J | Первая шкала P/J |
| 2 | E / I | E или I | Дихотомия Экстраверсия/Интроверсия |
| 3 | T / F | T или F | Дихотомия Мышление/Чувства |
| 4 | J / P | J или P | Вторая шкала J/P |
| 5 | N / S | N или S | Дихотомия Интуиция/Ощущения |

**Важно:** В тесте присутствуют две шкалы P/J — в вопросах №1 и №4. Это сделано специально для определения срединного типа W.

---

## 2️⃣ Логика определения итогового типа

Итоговая комбинация формируется **строго в порядке:**

```
[E/I] – [N/S] – [T/F] – [J/P/W]
```

### Пояснение по каждому блоку:

| Позиция | Блок | Источник данных | Пример значения |
|---------|------|-----------------|-----------------|
| **1** | **E/I** | Ответ на вопрос №2 | E или I |
| **2** | **N/S** | Ответ на вопрос №5 | N или S |
| **3** | **T/F** | Ответ на вопрос №3 | T или F |
| **4** | **J/P/W** | Ответы на вопросы №1 и №4 | J, P или W |

---

## 3️⃣ Правила для последней буквы (J/P/W)

Последняя позиция определяется на основе **сравнения ответов на вопросы №1 и №4**.

### Таблица правил:

| Ответ на вопрос №1 | Ответ на вопрос №4 | Итоговая буква | Логика |
|-------------------|-------------------|----------------|--------|
| J | J | **J** | Оба ответа совпадают → используем эту букву |
| P | P | **P** | Оба ответа совпадают → используем эту букву |
| J | P | **W** | Ответы расходятся → срединный тип |
| P | J | **W** | Ответы расходятся → срединный тип |

**Ключевое правило:** 
- Если ответы на вопросы №1 и №4 **совпадают** → используется **эта же буква** (J или P)
- Если ответы на вопросы №1 и №4 **расходятся** → используется **W** (срединный тип)

---

## 5️⃣ Привязка к базе отчётов

После формирования итогового типа (например: `ESTJ`, `ESTW`, `INFP`):

1. Система находит соответствующий отчёт из базы данных по **точному совпадению** `resultIndex`
2. Отчёт автоматически выдаётся пользователю
3. Если отчёт для данного типа не найден, система должна обработать ошибку соответствующим образом

## 6️⃣ Алгоритм в коде (псевдокод)

```
ФУНКЦИЯ resolveFreeResult(answers, config):
  // Сохранить ответы по вопросам 1–5
  answer_q1 = answers[1]  // 'P' | 'J'
  answer_q2 = answers[2]  // 'E' | 'I'
  answer_q3 = answers[3]  // 'T' | 'F'
  answer_q4 = answers[4]  // 'J' | 'P'
  answer_q5 = answers[5]  // 'N' | 'S'
  
  // Построить тип по позициям
  Type[0] = answer_q2              // E/I из вопроса 2
  Type[1] = answer_q5              // N/S из вопроса 5
  Type[2] = answer_q3              // T/F из вопроса 3
  Type[3] = resolve_JPW(answer_q1, answer_q4)  // J/P/W из вопросов 1 и 4
  
  // Функция resolve_JPW
  ЕСЛИ answer_q1 === answer_q4:
    ВЕРНУТЬ answer_q1  // J или P
  ИНАЧЕ:
    ВЕРНУТЬ 'W'        // Срединный тип
  
  // Найти отчёт по полученной комбинации
  resultIndex = Type[0] + Type[1] + Type[2] + Type[3]
  
  // Показать пользователю
  ВЕРНУТЬ resultIndex
КОНЕЦ ФУНКЦИИ
```

## 7️⃣ Важные замечания

### 7.1 Порядок важен
Порядок букв в результате **строго фиксирован**: `[E/I][N/S][T/F][J/P/W]`

Нельзя менять порядок позиций, так как это нарушит соответствие с базой отчётов.

### 7.2 Срединный тип W
Тип **W** появляется только в последней позиции и только когда ответы на вопросы №1 и №4 расходятся.

### 7.3 Валидация ответов
Перед расчётом результата необходимо убедиться, что:
- Все 5 вопросов имеют ответы
- Ответы соответствуют допустимым значениям для каждого вопроса
- Типы ответов корректны (например, вопрос №1 не может содержать 'E' или 'I')

### 7.4 Конфигурация через resultMapping
В реальной реализации используется конфигурация `resultMapping`, которая определяет:
- `position1.from` → номер вопроса для позиции 1 (обычно 2)
- `position2.from` → номер вопроса для позиции 2 (обычно 5)
- `position3.from` → номер вопроса для позиции 3 (обычно 3)
- `position4.from` → массив номеров вопросов для позиции 4 (обычно [1, 4])

Это позволяет гибко настраивать маппинг без изменения кода.

## 8️⃣ Возможные результаты

### Всего возможных комбинаций:

- **Без W:** 2 × 2 × 2 × 2 = **16 типов** (ESTJ, ESTP, ESFJ, ESFP, ENTJ, ENTP, ENFJ, ENFP, ISTJ, ISTP, ISFJ, ISFP, INTJ, INTP, INFJ, INFP)
- **С W:** 2 × 2 × 2 × 1 = **8 типов** (ESTW, ESFW, ENTW, ENFW, ISTW, ISFW, INTW, INFW)

**Итого:** 16 + 8 = **24 возможных типа личности** для FREE теста.
